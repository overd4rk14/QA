## HTTPS-протокол: этапы установления защищённого соединения
**Источник:** https://help.sweb.ru/protokol-https-chto-eto-princip-ego-raboty_1335.html#3

1. **Браузер инициирует запрос**
   - **Цель:** Запустить защищённое соединение вместо открытого HTTP
   - **Устраняет риск:** Использование незащищённого протокола, уязвимого для перехвата

2. **Сервер отправляет сертификат**
   - **Цель:** Передать браузеру подтверждение подлинности и публичный ключ сервера
   - **Устраняет риск:** Подмена сервера

3. **Браузер проверяет сертификат**
   - **Цель:** Убедиться в подлинности сервера перед установлением соединения
   - **Устраняет риск:** MITM-атаки, фишинг, использование поддельных или истёкших сертификатов

4. **Браузер генерирует и отправляет сеансовый ключ**
   - **Цель:** Безопасно обменяться ключом для шифрования всех последующих данных
   - **Устраняет риск:** Перехват сеансового ключа (он передаётся в зашифрованном виде)

5. **Сервер расшифровывает сеансовый ключ**
   - **Цель:** Синхронизировать браузер и сервер, дать обоим один и тот же сеансовый ключ
   - **Устраняет риск:** Несанкционированный доступ к ключу (только сервер может его расшифровать)

6. **Установление защищённого канала**
   - **Цель:**симметричное шифрование для безопасного обмена данными
   - **Устраняет риск:** Неэффективность асимметричного шифрования (симметричное работает быстрее)

7. **Обмен зашифрованными данными**
   - **Цель:** Передавать логины, пароли, данные карт и другую конфиденциальную информацию безопасно
   - **Устраняет риск:** Перехват данных, подделка, MITM-атаки (всё зашифровано сеансовым ключом)

8. **Завершение соединения**
   - **Цель:** Безопасно закрыть сеанс с инвалидацией сеансового ключа
   - **Устраняет риск:** Воспроизведение старых сеансов (каждый новый сеанс получит новый ключ)

## Анализ этапов установления HTTPS-соединения на практике

### Анализируемый сайт: bb.kai.ru

**Параметры соединения:**
- **Версия TLS:** 1.3
- **Выдан:** GlobalSign GCC R3 DV TLS CA 2020
- **Действителен:** 30 апреля 2025 — 1 июня 2026 г.
- **Шифр:** AES_256_GCM (современный, безопасный)
- **Обмен ключами:** X25519 (эллиптическая криптография, быстрый и устойчивый)

###  Скрытые этапы

| Этап | Почему скрыт | Что происходит внутри |
|------|--------------|----------------------|
| TCP Handshake (SYN, SYN-ACK, ACK) | Низкоуровневый сетевой процесс | Установка базового соединения между браузером и сервером перед началом TLS |
| Client Hello / Server Hello | Детали согласования скрыты | Браузер предлагает список поддерживаемых версий TLS и шифров, сервер выбирает лучший (результат виден — TLS 1.3) |
| Обмен ключами (X25519) | Криптографические вычисления скрыты | Клиент и сервер вычисляют общий секретный ключ без передачи его по сети напрямую |
| Проверка цепочки сертификатов | Выполняется автоматически браузером | Браузер проверяет, что сертификат подписан одним из доверенных корневых сертификатов своего хранилища |

## Шпаргалка по HTTP кодам состояния для тестирования

**Источник:** https://sky.pro/wiki/html/kody-sostoyaniya-http-chto-oni-znachat/

### 1xx Информационные

| Код | Что видит пользователь | Что происходит на самом деле | Как тестировать | Куда эскалировать |
|-----|-------------------------|------------------------------|----------------|-------------------|
| 100 | Страница продолжает загружаться | Сервер получил заголовки, клиент должен продолжить отправку тела запроса | Проверить в DevTools Network, отправить POST с большим телом | Бэкенд |
| 101 | Соединение переключается на другой протокол | Сервер переключается на протокол из заголовка Upgrade (например, WebSocket) | Использовать WebSocket соединение, проверить в DevTools | Бэкенд |
| 102 | Сервер обрабатывает запрос, ответ не готов | Сервер получил запрос и обрабатывает его, но ответ пока не готов | Длительные операции, проверить задержку в DevTools | Бэкенд |
| 103 | Предварительная информация для клиента | Сервер отправляет подсказки для оптимизации загрузки ресурсов | Проверить заголовки Link в DevTools | Фронтенд/Бэкенд |

### 2xx Успешные

| Код | Что видит пользователь | Что происходит на самом деле | Как тестировать | Куда эскалировать |
|-----|-------------------------|------------------------------|----------------|-------------------|
| 200 | Страница загружается нормально | Запрос успешно обработан, возвращён контент | Отправить GET запрос, проверить ответ в DevTools | Бэкенд |
| 201 | Ресурс создан (например, после регистрации) | Запрос выполнен, новый ресурс создан | Отправить POST для создания, проверить Location header | Бэкенд |
| 202 | Запрос принят на обработку | Запрос принят, но обработка ещё не завершена | Отправить асинхронный запрос, проверить статус | Бэкенд |
| 204 | Нет контента, но запрос успешен | Сервер обработал запрос, но не возвращает тело (например, DELETE) | Отправить DELETE, проверить отсутствие тела ответа | Бэкенд |
| 206 | Возвращена часть ресурса | Сервер возвращает только часть ресурса из-за заголовка Range | Отправить GET с Range header, проверить частичный контент | Бэкенд |

### 3xx Перенаправление

| Код | Что видит пользователь | Что происходит на самом деле | Как тестировать | Куда эскалировать |
|-----|-------------------------|------------------------------|----------------|-------------------|
| 301 | Страница перенаправлена навсегда | Ресурс перемещён на новый URL, браузер кэширует | Проверить Location header, повторный запрос идёт напрямую | Фронтенд/Бэкенд |
| 302 | Страница перенаправлена временно | Ресурс временно доступен по другому URL | Проверить Location, браузер не кэширует | Бэкенд |
| 303 | Ответ по другому URL с GET | Ответ на запрос находится по другому URL, использовать GET | Проверить Location и метод запроса | Бэкенд |
| 304 | Страница не изменилась, используется кэш | Ресурс не модифицирован, браузер использует кэшированную версию | Проверить If-Modified-Since header | Бэкенд |
| 307 | Временное перенаправление с сохранением метода | Временное перенаправление, сохраняет метод запроса | Проверить Location и метод (POST остаётся POST) | Бэкенд |
| 308 | Постоянное перенаправление с сохранением метода | Постоянное перенаправление, сохраняет метод запроса | Проверить Location и метод, браузер кэширует | Бэкенд |

### 4xx Ошибки клиента

| Код | Что видит пользователь | Что происходит на самом деле | Как тестировать | Куда эскалировать |
|-----|-------------------------|------------------------------|----------------|-------------------|
| 400 | Ошибка запроса, страница не загружается | Синтаксическая ошибка в запросе | Отправить malformed запрос, проверить в DevTools | Фронтенд |
| 401 | Требуется авторизация | Запрос требует аутентификации | Доступ к защищённому ресурсу без токена | Бэкенд |
| 403 | Доступ запрещён | Сервер понял запрос, но отказывает в выполнении | Доступ к ресурсу без прав, проверить роли | Бэкенд |
| 404 | Страница не найдена | Запрашиваемый ресурс не существует | Перейти на несуществующий URL | Бэкенд |
| 405 | Метод не разрешён | Метод запроса известен, но отключён | Отправить запрещённый метод (например, PUT на GET-only) | Бэкенд |
| 406 | Контент неприемлем | Сервер не может предоставить контент по Accept заголовкам | Установить специфические Accept headers | Фронтенд/Бэкенд |
| 409 | Конфликт с текущим состоянием | Запрос конфликтует с состоянием ресурса | Параллельное редактирование, проверить версию | Бэкенд |
| 413 | Полезная нагрузка слишком большая | Размер запроса превышает лимит | Отправить большой файл, проверить лимиты | DevOps |
| 415 | Неподдерживаемый тип медиа | Формат данных не поддерживается | Отправить неподдерживаемый Content-Type | Фронтенд |
| 422 | Непроцессируемый объект | Запрос корректен, но содержит семантические ошибки | Отправить валидный JSON с ошибками валидации | Бэкенд |
| 429 | Слишком много запросов | Превышен лимит запросов (защита от DoS) | Отправить много запросов быстро, проверить Retry-After | DevOps |

| Код | Что видит пользователь | Что происходит на самом деле | Как тестировать | Куда эскалировать |
|-----|-------------------------|------------------------------|----------------|-------------------|
