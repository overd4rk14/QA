## Ozon
### Гипотеза о типе БД
Я считаю, что для Ozon основной тип БД - гибридный. Реляционные БД  подходят для пользователей, заказов и платежей, где важна целостность. Нереляционные - для каталогов товаров и чатов, где нужна гибкость и масштабируемость.

### Разбиение сервиса на ключевые функции
1. **Управление каталогом товаров**: Данные - информация о товарах (название, цена, описание, категории, атрибуты). Тип БД: Нереляционная (документо-ориентированная), так как товары имеют переменную структуру атрибутов, и нужна гибкость для добавления новых полей без схемы.
2. **Обработка заказов и платежей**: Данные - заказы, пользователи, адреса доставки, платежные данные. Тип БД: Реляционная, для обеспечения транзакций, связей между таблицами (пользователь-заказ-товар) и целостности данных.
3. **Чат и поддержка**: Данные - сообщения, история чатов, файлы. Тип БД: Нереляционная (ключ-значение или документная), для хранения неструктурированных данных с высокой скоростью чтения/записи.


### Идеи тест-кейсов для проверки типов БД
1. **Проверка целостности данных в реляционной БД**: Создать заказ с несколькими товарами, проверить, что все связи (пользователь-заказ-товар) сохраняются корректно при обновлении или удалении. Ожидание: данные остаются консистентными.
2. **Проверка производительности нереляционной БД**: Загрузить большой объем данных в каталог товаров и выполнить поиск по нестандартным атрибутам. Ожидание: быстрый отклик без ошибок.

### Проверка гипотезы через поиск в интернете
Поиск: "Ozon tech stack database". Результаты показывают, что в Ozon используется гибридный подход к хранению и обработке данных: PostgreSQL с кастомным пулером соединений PgDoorman отвечает за транзакционные данные (заказы, пользователи), ClickHouse — за аналитику в реальном времени, а Cassandra — за высоконагруженные записи (корзина, сессии). Потоковую передачу данных обеспечивает Kafka, оркестрацию ETL-пайплайнов — Airflow, распределённые SQL-запросы к Data Lake — Trino, кэширование — Redis и Memcached, долговременное хранение сырых данных — Hadoop (HDFS), а визуализацию и BI-дашборды — внутренний портал Lusca.
Источник: https://github.com/ozontech
