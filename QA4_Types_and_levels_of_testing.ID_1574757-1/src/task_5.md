| № | Группа проверок | Описание проверки |
|----|----------------|-------------------|
| **1** | **Поле ввода названия магазина (SC21-01)** | |
| 1.1 | Ввод символов | Поле принимает буквы, цифры, пробелы и спецсимволы (например: “Магазин_123”) |
| 1.2 | Плейсхолдер | При пустом поле отображается плейсхолдер с текстом “Введите название магазина” |
| 1.3 | Исчезновение плейсхолдера | Плейсхолдер скрывается при вводе первого символа |
| 1.4 | Минимальная длина | При вводе 1 символа поле подсвечивается как невалидное |
| 1.5 | История поиска | При фокусе на поле появляется выпадающий список с 5 последними введёнными названиями |
| 1.6 | Очистка истории | Возможность очистить историю поиска через отдельную кнопку |
| **1.7** | **История поиска (БД)** | **После выполнения нескольких поисковых запросов проверить, что тексты запросов и/или ID найденных магазинов сохраняются в соответствующей таблице базы данных.** |
| **1.8** | **Отображение истории (БД)** | **Проверить, что при повторном открытии поля ввода данные для подсказок (история) корректно загружаются из БД.** |
| **2** | **Поле выбора типа магазина (SC21-02)** | |
| 2.1 | Выпадающий список | При нажатии на поле открывается список с тремя опциями |
| 2.2 | Одиночный выбор | Можно выбрать только один вариант из списка |
| 2.3 | Плейсхолдер | При отсутствии выбора отображается “Выберите тип магазина” |
| 2.4 | Отображение выбранного | После выбора в поле отображается выбранный тип |
| 2.5 | Сброс выбора | Крестик в поле сбрасывает выбор, восстанавливается плейсхолдер |
| **2.6** | **Проверка данных в БД** | **Убедиться, что справочник типов магазинов в БД строго соответствует списку в интерфейсе: "Продуктовые", "Стройматериалы", "Аптеки".** |
| **3** | **Кнопка «Поиск» (SC21-03)** | |
| 3.1 | Активность кнопки | Кнопка активна, если заполнено хотя бы одно поле (название ≥ 2 символов ИЛИ выбран тип) |
| 3.2 | Визуальное состояние неактивной кнопки | Неактивная кнопка имеет серый цвет и не реагирует на нажатие |
| 3.3 | Успешный поиск | При успешном поиске на карте появляются флажки (максимум 20) |
| 3.4 | Внешний вид флажка | Флажок отображает иконку типа магазина и название при тапе |
| 3.5 | Отсутствие результатов | Если результатов нет, выводится сообщение “Ничего не найдено” |
| **3.6** | **Запрос к БД** | **При нажатии кнопки «Поиск» проверить логи или перехватить запрос к БД. Убедиться, что формируется корректный SQL-запрос, который учитывает введённое название (частичное совпадение) и выбранный тип.** |
| **3.7** | **Верификация результатов** | **Сравнить количество и идентификаторы объектов, возвращённых в результате SQL-запроса, с количеством и позициями флажков, отображённых на карте. Они должны совпадать.** |
| **4** | **Кнопка «Построить маршрут» (SC21-04)** | |
| 4.1 | Активность при выборе флажка | Кнопка активна, если выбран один флажок на карте |
| 4.2 | Активность при выборе типа | Если выбран только тип, кнопка строит маршрут до ближайшего магазина этого типа |
| 4.3 | Определение “ближайшего” | Маршрут строится до магазина в радиусе 5 км |
| 4.4 | Изменение состояния кнопки | При активации текст меняется на “Отменить маршрут”, фон — на #586d73 |
| 4.5 | Отмена маршрута | При нажатии на “Отменить маршрут” маршрут исчезает, кнопка возвращается в исходное состояние |
| **4.6** | **Данные для маршрута** | **При построении маршрута проверить, что в сервис построения маршрута передаётся корректный ID магазина, полученный из БД, а не только его координаты или название.** |
| **5** | **Область карты (SC21-05)** | |
| 5.1 | Расположение карты | Карта занимает нижнюю половину экрана, над ней — форма поиска |
| 5.2 | Адаптивность | На устройствах с разным разрешением карта сохраняет пропорции |
| 5.3 | Построение маршрута | Маршрут отображается синей линией с указанием времени и расстояния |
| 5.4 | Кнопка отмены маршрута | В режиме маршрута на карте появляется кнопка “Отмена” для его отмены |
| 5.5 | Зум и скролл карты | Карта реагирует на жесты масштабирования и прокрутки |
| **6** | **Общее поведение приложения (SC21-06)** | |
| 6.1 | Восстановление маршрута | При перезапуске приложения незавершённый маршрут отображается автоматически |
| 6.2 | Хранение данных | **Проверить БД или локальное хранилище  на наличие записи о незавершённом маршруте (ID маршрута, состояние, временная метка). Данные должны сохраняться.** |
| 6.3 | Геолокация при низком заряде | При уровне заряда ≤ 15% используется приблизительная геолокация |
| **6.4** | **Логика отключения геолокации** | **Проверить логи приложения или системные настройки. При падении заряда батареи ниже порога должна появляться запись о переключении режима геолокации с точного на приближённый или кешированный.** |
